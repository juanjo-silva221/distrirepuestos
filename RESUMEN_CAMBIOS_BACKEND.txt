2. ARQUITECTURA DEL BACKEND

Antes el backend tenia principalmente:

controller -> repository -> base de datos

Ahora quedo organizado por capas:

controller -> service -> repository -> base de datos

Esto mejora la estructura porque los controladores ya no contienen la logica de negocio.
Los controladores solo reciben las peticiones HTTP y llaman a los servicios.


3. CAPA SERVICE AGREGADA

Se agrego el paquete:

src\main\java\com\distrirepuestos\service

Servicios creados:

ProductService.java
AuthService.java
OrderService.java
ContactService.java

Responsabilidades:

ProductService:
- Listar productos.
- Buscar producto por ID.
- Crear productos.
- Actualizar productos.
- Eliminar productos.

AuthService:
- Registrar usuarios.
- Validar login.
- Cifrar contrasenas nuevas con BCrypt.
- Permitir login de usuarios antiguos con contrasena sin cifrar y actualizarla a BCrypt.

OrderService:
- Crear pedidos.
- Generar codigo de pedido.
- Calcular total del pedido.
- Guardar items del pedido.
- Consultar pedidos por correo.

ContactService:
- Guardar mensajes de contacto.
- Asignar fecha automaticamente.


4. CONTROLADORES REFACTORIZADOS

Se actualizaron estos controladores:

ProductController.java
AuthController.java
OrderController.java
ContactController.java

Ahora los controladores delegan la logica a los servicios.

Ejemplo:

ProductController usa ProductService.
AuthController usa AuthService.
OrderController usa OrderService.
ContactController usa ContactService.


5. CRUD DE PRODUCTOS

Se agregaron endpoints completos para productos:

GET    /api/products
GET    /api/products/{id}
POST   /api/products
PUT    /api/products/{id}
DELETE /api/products/{id}

Esto permite listar, consultar, crear, actualizar y eliminar productos.


6. MANEJO GLOBAL DE ERRORES

Se agrego el paquete:

src\main\java\com\distrirepuestos\exception

Archivos creados:

ApiError.java
GlobalExceptionHandler.java
ResourceNotFoundException.java

Esto permite devolver errores mas ordenados cuando algo falla, por ejemplo:

- Producto no encontrado.
- Datos invalidos.
- Usuario o contrasena incorrectos.
- Error interno del servidor.


7. CIFRADO DE CONTRASENAS

Se agrego:

PasswordConfig.java

Ubicacion:

src\main\java\com\distrirepuestos\config\PasswordConfig.java

Tambien se agrego esta dependencia en pom.xml:

spring-security-crypto

Con esto, las contrasenas nuevas se guardan cifradas con BCrypt.


8. CONFIGURACION UTF-8

Se agrego configuracion UTF-8 en pom.xml:

project.build.sourceEncoding=UTF-8

Esto ayuda a evitar problemas con textos en espanol.
